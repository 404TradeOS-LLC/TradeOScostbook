## The integration (CSS-only, ~15 line diff)

Your `--theme inline` block, component classes, and shadcn component code **do not change at all** — every `bg-primary`, `text-muted-foreground`, etc. across the whole app keeps working exactly as-is. Only the *values* those tokens resolve to change, plus two font-mapping lines.

### 1. Turn on the font Space Grotesk you already loaded

```diff
  --font-sans: var(--font-inter), "Avenir Next", "Segoe UI", system-ui, sans-serif;
  --font-mono-base: var(--font-ibm-plex-mono), "SFMono-Regular", "SF Mono", "Consolas", monospace;
```
stays the body/mono default — but in the `@theme inline` block:
```diff
- --font-heading: var(--font-sans);
+ --font-heading: var(--font-space-grotesk), var(--font-sans);
```
Then apply `font-heading` (Tailwind utility, already generated by this token) to your `h1`–`h3` / page-title components. Zero new dependencies — the variable is already in `layout.tsx`.

### 2. Recolor `--primary` and `--ring` to copper (light mode)

```diff
  --primary: oklch(0.205 0 0);
  --primary-foreground: oklch(0.985 0 0);
```
→
```diff
- --primary: oklch(0.205 0 0);
- --primary-foreground: oklch(0.985 0 0);
+ --primary: #B87333;              /* copper — the one accent, per Brand Bible */
+ --primary-foreground: #FFFFFF;
```
```diff
- --ring: oklch(0.708 0 0);
+ --ring: #B87333;
```
```diff
- --sidebar-primary: oklch(0.205 0 0);
- --sidebar-primary-foreground: oklch(0.985 0 0);
+ --sidebar-primary: #B87333;
+ --sidebar-primary-foreground: #FFFFFF;
+ --sidebar-ring: #B87333;
```

**Do not touch** `--secondary`, `--accent`, `--muted`, `--chart-*`, or `--destructive` — those stay neutral/shadcn-default. Per the Bible's hard rule, copper appears in ≤3 semantic roles (primary action, ring/focus, active sidebar item) — this diff is exactly those three and nothing else.

### 3. Same three overrides in `.dark`

```diff
  --primary: oklch(0.922 0 0);
  --primary-foreground: oklch(0.205 0 0);
```
→
```diff
- --primary: oklch(0.922 0 0);
- --primary-foreground: oklch(0.205 0 0);
+ --primary: #E8C99A;              /* copper-light — better contrast on dark surfaces */
+ --primary-foreground: #1c0f04;
```
```diff
- --ring: oklch(0.556 0 0);
+ --ring: #B87333;
```
```diff
- --sidebar-primary: oklch(0.488 0.243 264.376);   /* currently an unrelated blue */
- --sidebar-primary-foreground: oklch(0.985 0 0);
+ --sidebar-primary: #E8C99A;
+ --sidebar-primary-foreground: #1c0f04;
```

### 4. Corner radius (optional, low-risk)

Current `--radius: 0.625rem` (10px) already matches the Bible's card radius almost exactly — **no change needed here**. Leave as-is; this is a case where the existing engineering already matches the approved system.

### 5. Status colors (if/where the app has success/error states)

Wherever the app renders a status pill/badge (project status, invoice status, membership state), map to:
- success/online → `#0E9F6E` (Bible's `--success-600`, AA on white)
- error/destructive → keep the existing `--destructive` token (already reasonable) — no change needed
- warning (if any) → `#D9891F`

This is additive — if no such tokens exist yet in `globals.css`, add them under a new `--color-status-*` block rather than repurposing `--accent`.
