R1Baseline — wide format, 3:1 (current approved)
Reference point, unchanged, for comparison against every refinement below.
R2Tighter chamfer, thinner rivets — refined proportions
Rivets reduced 7px → smaller, quieter presence; chamfer tightened to 10px for a crisper machined edge; tick-rulers removed for a cleaner read at small sizes (app icon, favicon-adjacent contexts).
R3Deeper engraving — heavier bevel & drop
Stacked drop-shadow (dark below, light rim above) reads as a deeper cut into the plate — more "recessed into metal," less "printed on metal." Best at large/hero scale where the depth registers.
R4Compressed square format — for app icon / avatar contexts
A distinct square construction — not the wide nameplate squeezed into a box. "404" mono + copper "OS" stacked, same fasteners/chamfer language, purpose-built for icon/favicon/avatar slots where the full wordmark can't fit.
R5Single-line compact lockup — for nav bars & headers
No plate/rivets at this scale — a thin steel "tab" accent on the left stands in for the fastener language, keeping nav bars uncluttered while still nodding to the fabricated-metal identity.
R6Vertical/stacked format — for narrow surfaces (business card, hang tag)
Divider rotates from vertical to horizontal for the stacked lockup — same construction logic, adapted rather than shrunk, so it never looks like an afterthought crop of the horizontal mark.
R7Low-contrast field variant — for on-photo / on-fabric use
Flat, no plate, no bevel — for contexts where the metal treatment would fight the substrate (screen-printed on dark fabric, debossed on leather, embroidered patches at small scale where gradients don't survive).
R8Extreme small-size legibility test (32px height)
The "OS" monogram favicon holds up down to 16px — the threshold for a browser tab icon. Confirms the mark's minimum viable size before the full nameplate/wordmark is required instead.
